# 안드로이드 앱으로 사용하기 (한글 가이드)

이 문서는 **Gemini 실시간 통역기**를 안드로이드폰에 앱(APK)으로 설치해서
쓰는 전체 과정을 설명합니다. 구조는 이렇습니다.

```
[ 안드로이드 앱(APK) ]  ⇄  wss(보안 웹소켓)  ⇄  [ 클라우드 백엔드 ]  ⇄  [ Gemini ]
   마이크/스피커                                  (API 키 보관)
```

API 키는 앱이 아니라 **클라우드 백엔드**에 보관됩니다(안전). 앱은 그 서버에
마이크 오디오를 보내고, 통역된 오디오를 받아서 재생만 합니다.

순서는 딱 3단계입니다.
**(1) 백엔드 클라우드 배포 → (2) APK 빌드 → (3) 폰에 설치하고 서버주소 입력**

---

## 1단계 — 백엔드를 클라우드에 배포 (Render, 무료)

1. 이 저장소를 GitHub에 올립니다(이미 올라가 있으면 생략).
2. <https://render.com> 가입 후 로그인.
3. **New > Blueprint** 클릭 → 이 저장소 선택.
   저장소의 `gemini_live_translator/render.yaml` 을 자동으로 인식합니다.
4. 배포 중 `GEMINI_API_KEY` 입력란이 나오면 본인의 Gemini API 키를 붙여넣습니다.
   (키 발급: <https://aistudio.google.com/apikey>)
5. 배포가 끝나면 `https://gemini-live-translator.onrender.com` 같은 **공개 주소**가
   생깁니다. 이 주소를 적어두세요. 브라우저에서 그 주소 뒤에 `/api/health` 를
   붙여 열었을 때 `"status":"ok"` 가 보이면 정상입니다.
6. **접속 토큰(ACCESS_TOKEN) 복사** — Render가 강력한 토큰을 자동 생성합니다.
   서비스 → **Environment** 탭 → `ACCESS_TOKEN` 값을 복사해 두세요. (3단계 앱에서 입력)

> 🔒 **왜 토큰이 필요한가:** 서버 주소만 알면 누구나 접속해 통역을 쓸 수 있고
> 그 비용이 사장님 Gemini 키로 과금됩니다. 토큰을 설정하면 토큰을 가진 사장님
> 앱만 서버를 쓸 수 있습니다. 토큰은 깃허브에 올라가지 않으므로(서버·폰에만
> 저장) 공개 저장소여도 안전합니다.

> 무료 플랜은 한동안 안 쓰면 잠들었다가 첫 접속 때 10~30초 깨어나는 지연이
> 있습니다. 바로 반응이 없으면 잠깐 기다렸다가 다시 시도하세요.

---

## 2단계 — APK 빌드 (GitHub Actions, 안드로이드 스튜디오 불필요)

이 저장소에는 APK를 자동으로 만들어 주는 GitHub Actions가 들어 있습니다.

1. GitHub 저장소의 **Actions** 탭으로 갑니다.
2. 왼쪽에서 **Build Android APK** 워크플로 선택 → **Run workflow** 클릭.
3. 5~10분 뒤 빌드가 끝나면, 그 실행 결과 페이지 맨 아래 **Artifacts** 에서
   `gemini-translator-debug-apk` 를 내려받습니다(압축 안에 `app-debug.apk`).

> 앱에 클라우드 주소를 미리 박아두고 싶으면, 빌드 전에
> `gemini_live_translator/static/config.js` 의
> `window.DEFAULT_SERVER_URL` 에 1단계의 주소를 적어두면 됩니다.
> (비워두면 앱 안에서 직접 입력 — 한 번 입력하면 폰에 기억됩니다.)

### 안드로이드 스튜디오로 직접 빌드하는 경우(선택)

```bash
cd gemini_live_translator
npm install
npx cap add android
node scripts/patch-android.mjs   # 마이크 권한 주입
npx cap sync android
npx cap open android             # 안드로이드 스튜디오에서 Run
```

## 3단계 — 폰에 설치하고 사용

1. 받은 `app-debug.apk` 를 폰으로 옮겨 실행합니다.
   "출처를 알 수 없는 앱 설치"를 허용해야 설치됩니다(설정에서 일시 허용).
2. 앱을 열고, 마이크 권한 요청이 뜨면 **허용**.
3. **Server URL** 칸에 1단계의 클라우드 주소(`https://...onrender.com`)를 넣습니다.
4. **Access Token** 칸에 2단계 6번에서 복사한 토큰을 붙여넣고 **Save**.
5. **Languages → Language B** 에 **내가 듣고 싶은 언어**를 고릅니다(출력 언어).
   기본 통역 모델(`gemini-3.5-live-translate-preview`)은 상대가 어떤 언어로
   말해도 **자동 감지(70+ 언어)** 해서 B 언어 음성으로 통역합니다.
   (목록의 8개 언어 외에도 코드에 추가하면 더 늘릴 수 있습니다.)
6. **Start Translating** 를 누르고 말하면, 통역된 음성이 스피커로 나오고
   화면에 원문/번역 자막이 실시간으로 뜹니다.

> **이어폰(헤드셋) 사용을 권장합니다.** 스피커로 통역 음성을 크게 내보내면
> 마이크가 그 소리를 다시 잡아 되먹임(에코)이 생길 수 있습니다.

### 통역 중 편의 기능

- **🌐 자동 감지**: Languages의 첫 번째(A)를 "자동 감지"로 두면, 상대가 어떤
  언어로 말해도 두 번째(B) 언어로 통역합니다(여러 언어가 섞인 자리에서 유용).
- **목소리/빠르기**: 남성·여성 음성을 고르고, Speed 슬라이더로 통역 음성의
  재생 속도(0.5×~1.5×)를 실시간 조절합니다.
- **⏸️ 일시정지**: 마이크 전송을 잠시 멈춰 중요한 부분을 확인하고, ▶️ 재개로
  이어갑니다(연결은 유지).
- **🔁 다시 듣기**: 방금 통역된 문장의 음성을 다시 재생합니다.
- **🗣️ 발음**: 방금 통역된 외국어 문장을 **로마자** 또는 **한글** 발음으로
  보여줍니다.

### 🛡️ 리스크 감지 (협상 코파일럿)

- 컨트롤 패널의 **리스크 감지** 스위치를 켜면, 발화가 **끝난 뒤** 별도로 분석해
  계약·금전·선결제·보증·숫자/단위·예절 측면의 위험을 감지합니다.
- 자막 위에 **확인 알림 카드**(참고/확인필요/주의 3단계 색상)와 **추천 질문**이
  뜹니다. 추천 질문은 **복사** 버튼으로 바로 가져올 수 있습니다.
- **분야** 칸에 `oil trading`, `알루미늄 수출` 등을 적으면 그 분야 전문용어
  (LOI·SCO·POP·SGS·Platts·CIF/FOB 등)를 더 잘 잡아냅니다.
- 통역 음성·자막이 모두 나온 *뒤에* 분석이 따라붙으므로 **통역 속도에는 영향이
  없습니다.** 감지된 위험은 회의 노트와 내보내기 파일에도 함께 저장됩니다.

> 분석은 발화마다 1회 호출되어 약간의 API 비용이 듭니다. 필요할 때만 켜세요.

### 회의 노트 / 요약 사용법

화면 아래 **Meeting Notes** 영역에서:
- 통역이 진행되면 발화가 **시간순으로 자동 기록**됩니다(원문 → 번역). 기록은
  폰에 저장되어 앱을 닫아도 유지됩니다.
- **Summary in** 에서 요약 언어를 고르고 **Summarize** 를 누르면, 지금까지의
  회의 내용을 **한 줄 요약 / 핵심 논의 / 결정사항 / 액션아이템** 으로 정리해
  보여줍니다. 회의 중간중간 눌러 최신 요약으로 갱신할 수 있습니다.
- **저장**: **MD / Word / PDF** 버튼으로 노트+요약을 파일로 내려받습니다.
  (PDF는 한글 폰트 내장 — 한국어·영어는 완벽, 아랍어 등 다른 문자는 Word 권장.)
- **💬 대화 내용 질문**: 입력창에 질문하면 저장된 대화 내용을 근거로 답합니다.
  (예: "결제 조건이 어떻게 정해졌지?")
- **New** 를 누르면 현재 노트를 지우고 새 회의를 시작합니다.

### 🔎 의미 확인 (발음 어눌할 때)

컨트롤 패널의 **의미 확인** 스위치를 켜면, 발음이 어눌해 잘못 통역된 것 같은
발화를 **문맥을 보고** 감지해 *"혹시 이런 뜻이었나요?"* + **교정 번역**을
자막 위에 띄웁니다. 리스크 감지와 **같은 1회 분석 호출**을 공유하므로 통역
속도에는 영향이 없습니다.

## 자주 막히는 부분

| 증상 | 원인 / 해결 |
| --- | --- |
| "Cannot reach server" | Server URL 오타, 또는 백엔드가 잠든 상태. 주소 확인 후 잠시 뒤 재시도. |
| "Unauthorized: invalid access token" | 앱의 Access Token이 Render의 `ACCESS_TOKEN`과 불일치. 값 다시 복사·입력. |
| "API key not configured" | Render 환경변수 `GEMINI_API_KEY` 미설정. Render 대시보드에서 추가. |
| 마이크가 안 잡힘 | 폰 설정 > 앱 > 권한 에서 마이크 허용 확인. |
| 소리는 나오는데 자막이 늦음 | 정상입니다. 자막은 음성보다 약간 늦게 스트리밍됩니다. |
| 통역이 끊김/되먹임 | 이어폰을 쓰세요(스피커 + 마이크 동시 사용 시 에코). |

자세한 백엔드/아키텍처 설명은 [`README.md`](./README.md) 를 참고하세요.
